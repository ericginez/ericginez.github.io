## Contexte

Le premier projet du parcours Data Engineer a pour objectif de mettre
l’apprenant dans les meilleures conditions pour réussir sa formation.

Il intervient avant les projets techniques et permet de construire un cadre de
travail cohérent pour l’ensemble du parcours.

Le programme officiel lui attribue une durée estimée de 13 heures et définit
une compétence cible principale :

```text
Définir le cadre de sa formation
```

Le projet porte sur trois dimensions complémentaires :

- la compréhension du fonctionnement du parcours ;
- la définition d’un objectif professionnel ;
- l’organisation concrète du travail et de l’apprentissage.

## Objectifs

Le projet devait permettre de :

- comprendre les modalités d’une formation OpenClassrooms ;
- identifier les principaux interlocuteurs et ressources disponibles ;
- formaliser un objectif professionnel ;
- analyser les treize projets du parcours ;
- repérer les projets les plus motivants et les difficultés anticipées ;
- définir un calendrier prévisionnel des évaluations ;
- choisir des méthodes de gestion du temps ;
- organiser la collaboration avec le mentor ;
- adopter une démarche d’apprentissage autonome ;
- utiliser l’intelligence artificielle de façon productive et critique.

## Périmètre du parcours

Le parcours Data Engineer est structuré autour de treize projets progressifs.

Il couvre notamment :

- l’analyse exploratoire de données ;
- SQL et les bases de données relationnelles ;
- la qualité et la gouvernance des données ;
- les bases NoSQL ;
- le machine learning ;
- la visualisation et la Business Intelligence ;
- le cloud et l’infrastructure ;
- le traitement de données en continu ;
- l’orchestration de pipelines ;
- les systèmes RAG ;
- l’automatisation d’architectures de données ;
- le passage d’un POC à un MVP ;
- la valorisation des compétences dans un portfolio.

Le projet 1 permet de prendre connaissance de cette progression avant le
démarrage des travaux techniques.

## Compréhension du fonctionnement de la formation

La phase de cadrage a permis d’identifier les principaux éléments du dispositif
pédagogique :

- une formation réalisée entièrement en ligne ;
- des projets encadrés par un mentor ;
- des soutenances ou sessions de bilan pour valider les projets ;
- des cours et ressources pédagogiques associés à chaque projet ;
- une communauté d’apprenants accessible à distance ;
- des outils d’aide, de documentation et de communication ;
- un tableau de bord permettant de suivre l’avancement ;
- des interlocuteurs pédagogiques disponibles en cas de difficulté.

Cette compréhension initiale permet d’anticiper les responsabilités de
l’apprenant et de mieux utiliser les ressources proposées.

## Définition de l’objectif professionnel

Le projet comprend la formalisation d’un objectif de reconversion vers le
métier de Data Engineer.

Cet objectif sert de fil directeur pour le parcours et permet de relier les
projets pédagogiques aux compétences attendues dans un contexte professionnel.

Les axes prioritaires identifiés sont notamment :

- consolider les compétences en programmation ;
- développer la maîtrise de SQL ;
- apprendre à construire et automatiser des pipelines ;
- comprendre les architectures de données ;
- acquérir une expérience pratique du cloud ;
- maîtriser les outils de versionnement et de collaboration ;
- produire des réalisations démontrables dans un portfolio.

## Analyse prévisionnelle des projets

L’ensemble des projets a été étudié afin d’identifier :

- les compétences déjà partiellement connues ;
- les sujets les plus motivants ;
- les domaines demandant un effort d’apprentissage plus important ;
- les dépendances entre les différents projets ;
- les périodes nécessitant davantage de temps ou d’accompagnement.

Les projets portant sur Python, SQL, les bases de données et Git ont été
identifiés comme des étapes particulièrement motivantes.

Les projets consacrés au cloud, à l’infrastructure et au traitement distribué
ont été identifiés comme des étapes plus exigeantes, nécessitant une préparation
et une expérimentation renforcées.

Cette analyse permet de répartir l’effort de manière réaliste et d’anticiper les
points de vigilance.

## Planification du parcours

Un calendrier prévisionnel a été établi pour les treize projets.

La planification prend en compte :

- la date de début de formation ;
- la date cible de fin de parcours ;
- la charge estimée de chaque projet ;
- le temps nécessaire au suivi des cours ;
- les recherches et expérimentations complémentaires ;
- les sessions de mentorat ;
- les soutenances et bilans ;
- une marge pour les corrections et les imprévus.

Les dates cibles servent de repères et peuvent être ajustées en fonction de la
complexité réelle des projets et des retours du mentor.

## Organisation hebdomadaire

Une organisation régulière a été définie afin de maintenir une progression
continue.

Elle repose sur plusieurs principes :

- réserver des plages horaires dédiées à la formation ;
- distinguer le temps consacré aux cours du temps consacré aux projets ;
- avancer par objectifs intermédiaires ;
- conserver une marge pour la résolution des difficultés ;
- documenter les décisions et les apprentissages ;
- réviser régulièrement les notions étudiées ;
- ajuster le planning en fonction de l’avancement réel.

Cette organisation vise à éviter les périodes de surcharge et à assurer une
progression durable pendant les neuf mois de formation.

## Méthodes de gestion du temps

Plusieurs pratiques ont été retenues pour structurer le travail.

### Méthode Pomodoro

Le travail est découpé en périodes de concentration suivies de pauses courtes.

Cette méthode facilite :

- le démarrage des tâches ;
- le maintien de l’attention ;
- la réduction de la fatigue ;
- le suivi du temps réellement consacré à une activité.

### Plans d’action et listes de tâches

Chaque projet peut être décomposé en actions de taille limitée :

```text
Comprendre le besoin
    ↓
Identifier les livrables
    ↓
Planifier les étapes
    ↓
Produire une première version
    ↓
Tester et corriger
    ↓
Documenter
    ↓
Préparer la soutenance
```

Cette décomposition rend l’avancement visible et facilite la priorisation.

### Objectifs SMART

Les objectifs sont formulés pour être :

- spécifiques ;
- mesurables ;
- atteignables ;
- réalistes ;
- définis dans le temps.

Cette méthode permet de transformer un projet général en résultats
intermédiaires vérifiables.

## Apprentissage assisté par l’intelligence artificielle

Le projet comprend une formation aux usages de l’intelligence artificielle dans
un contexte d’apprentissage.

Les usages étudiés sont organisés autour de six catégories.

### 1. Planifier son apprentissage

L’IA peut aider à :

- construire un emploi du temps ;
- répartir une charge de travail ;
- organiser des révisions ;
- transformer un objectif global en étapes intermédiaires.

### 2. Clarifier et approfondir un contenu

Elle peut être utilisée pour :

- reformuler une notion complexe ;
- expliquer un message d’erreur ;
- proposer des exemples ;
- comparer plusieurs approches ;
- faciliter l’accès à des ressources en langue étrangère.

### 3. Vérifier les informations

L’utilisation de l’IA doit être accompagnée d’une démarche critique :

- rechercher les sources d’origine ;
- vérifier les affirmations importantes ;
- comparer plusieurs références ;
- distinguer un fait d’une interprétation ;
- identifier les informations potentiellement obsolètes.

### 4. Synthétiser et structurer

L’IA peut aider à :

- résumer un document ;
- comparer plusieurs contenus ;
- structurer un rapport ;
- préparer un plan ;
- identifier les idées principales.

### 5. Vérifier ses acquis

Elle peut également servir à :

- générer des questionnaires ;
- proposer des exercices pratiques ;
- simuler des questions de soutenance ;
- expliquer les erreurs commises ;
- adapter la difficulté des exercices.

### 6. Finaliser un livrable

L’IA peut contribuer à :

- relire un document ;
- améliorer sa clarté ;
- vérifier l’orthographe ;
- proposer un retour critique ;
- préparer des éléments visuels ;
- contrôler la cohérence générale d’une présentation.

## Bonnes pratiques d’utilisation de l’IA

Trois règles principales structurent l’usage de l’intelligence artificielle.

### Formuler des demandes précises

Une demande utile précise :

- le contexte ;
- l’objectif ;
- les contraintes ;
- le format de réponse attendu ;
- le niveau de détail nécessaire.

### Conserver un esprit critique

Les réponses produites doivent être considérées comme des propositions à
évaluer.

Les éléments importants doivent être :

- compris ;
- vérifiés ;
- testés ;
- reformulés ;
- adaptés au contexte réel du projet.

### Maintenir une veille

Les outils et pratiques liés à l’intelligence artificielle évoluent rapidement.

Une veille régulière est nécessaire pour :

- suivre les nouvelles fonctionnalités ;
- identifier les limites des outils ;
- connaître les enjeux de confidentialité ;
- adapter ses méthodes de travail ;
- conserver des pratiques responsables.

## Résultats

Le projet a permis de disposer, dès le début du parcours, d’un cadre commun pour
piloter la formation.

Les principaux résultats sont :

- une vision globale des treize projets ;
- un objectif professionnel clairement formulé ;
- un planning prévisionnel ;
- une méthode de travail structurée ;
- des outils de collaboration identifiés ;
- des pratiques de gestion du temps sélectionnées ;
- des règles d’utilisation responsable de l’IA ;
- une première identification des risques d’apprentissage.

## Limites

Le projet présente plusieurs limites :

- le planning initial repose sur des estimations ;
- la complexité réelle de chaque projet n’est pas encore connue ;
- les dates peuvent évoluer selon les difficultés rencontrées ;
- les méthodes de travail doivent être ajustées à l’usage ;
- les outils d’intelligence artificielle évoluent rapidement ;
- la qualité des réponses de l’IA dépend fortement du contexte fourni ;
- l’IA ne remplace ni l’apprentissage, ni les tests, ni la validation humaine ;
- les documents personnels de cadrage ne sont pas publiés dans le portfolio.

## Évolutions possibles

Le cadre défini peut évoluer tout au long de la formation :

- comparer le planning prévisionnel à l’avancement réel ;
- ajuster les charges estimées après chaque projet ;
- enrichir les méthodes de révision ;
- formaliser les retours d’expérience ;
- compléter la veille sur les outils d’IA ;
- mesurer les gains de temps obtenus ;
- documenter les difficultés récurrentes ;
- améliorer la préparation des soutenances ;
- intégrer les réalisations validées au portfolio ;
- produire un bilan global à la fin du parcours.
